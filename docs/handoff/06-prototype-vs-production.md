# 06. Прототип против продакшена

Chappy Web v0.1 — это **frontend-only прототип**: нет бэкенда, нет сети (в коде ноль `fetch`/`axios`/`WebSocket`/`import.meta.env`), нет API-ключей. Все данные — seed-моки в `src/data/*`, состояние живёт в `zustand` и персистится в `localStorage` (ключ `chappy_v01_store`, version 4). Генерация и работа агентов имитируются `setTimeout`-таймерами, результат — случайная картинка из 13 статичных `jpg` в `public/trends/`.

Ключевая ценность прототипа для продакшена — **доменная модель (`src/types/index.ts`), формы store-экшенов, типизированные швы для замены (`services/generation.ts`, `features/agents/orchestration.ts`) и весь UI/UX**. Выбрасываются — тела моков (таймеры, `DEMO_POOL`, случайные ошибки), seed-данные и слой персистентности.

## Швы для замены (сохранить сигнатуры)

Прототип уже спроектирован под замену. Три чётких шва, документированных в самом коде:

1. **`services/generation.ts`** → реальный create-job + poll/stream. Сохранить `runGeneration(input: GenerationInput, h: GenerationHandlers): GenerationJob`. Весь Canvas зависит только от этого интерфейса.
2. **`features/agents/orchestration.ts`** → реальный оркестратор агентов. Сохранить экспорт `startRun/cancelRun/retryStep/requestRevision/approveRun/setForceErrorStep`.
3. **`store/useStore.ts` (persist)** → серверные CRUD. Формы экшенов и `partialize`-набор слайсов = готовый контракт таблиц/эндпоинтов.

## Что переиспользуется vs выбрасывается — кратко

**Переиспользовать (ценность прототипа):** весь `src/types/index.ts` (доменная модель); формы store-экшенов как контракт API; интерфейсы `generation.ts` и API `orchestration.ts`; типизированные `contextResolver.ts` / `agentContext.ts`; алгоритм `getAssetLineage`; модель сущностей Avatar/Voice/Style/Location; весь Canvas-слой; все страницы/компоненты/UX; i18n (`i18n/ru.ts`); паттерн честности `MockBadge` + `FuturePage` (явная маркировка демо/будущего функционала).

**Выбросить (throwaway):** тела моков (таймеры, `DEMO_POOL`, случайные ошибки/URL, canned-выводы шагов); seed-данные `src/data/*`; конфиг `localStorage`-персистентности и `merge`/`resetDemo`; демо-ЮKassa и кредиты как клиентское число; статичные картинки `public/trends/*` как «результаты генерации».
